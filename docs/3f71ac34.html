사용자 캡챠 인증 시스템

캡챠를 통해 사용자가 봇인지 확인할 수 있습니다. 이 시스템을 사용할 경우 인증된 사용자만 메시지를 전송할 수 있도록 제한합니다.

캡챠 인증 시스템과 함께 서버 설정에서 보안 설정 하는 것을 권장합니다(서버 보안 설정 )

/authorization_on

캡챠 인증 시스템을 구축합니다. 1번부터 버튼을 눌러야 하며 완료 메시지가 출력된 후 다음 단계의 버튼을 눌러야 합니다.

  1. 1번 버튼으로 ‘갈매기’라는 역할을 추가합니다. 만약 해당 이름의 역할이 존재한다면 갈매기 봇의 인증시스템이 그 역할을 사용하도록 설정합니다.
  1. 2번 현재 모든 사용자들에게 역할을 부여합니다. 과도한 api 호출 발생을 우려하여 1명당 1초의 시간이 걸립니다. 이 단계는 유일하게 건너뛰어도 됩니다.(대신 사용자가 메시지 전송을 원한다면 각자 캡챠 인증을 진행해야 합니다.)
  1. 3번 기존 채널의 권한 수정을 실행하면 인증된 사용자만 메시지가 전송이 가능하며, 기존의 일반 사용자가 메시지를 전송 허락하지 않은 채널은 그대로 권한을 못 보내도록 유지합니다.
  1. 4번 버튼으로 인증 채널이 추가됩니다. 갈매기 카테고리에 생성되도록 설정되어 있으며 관리자가 원할 경우 카테고리나 채널명을 변경하여도 무방합니다. 채널의 고유 ID로 관리하기 때문에 이름 변경이 기능에 영향을 주지 않습니다.

/change_authorization_message

사용자 인증 채널의 안내 문구 메시지를 삭제하고 새롭게 지정한 메시지로 출력합니다.

/authorization_off

캡챠 인증 시스템을 제거합니다. 1번부터 버튼을 눌러야 하며 완료 메시지가 출력된 후 다음 단계의 버튼을 눌러야 합니다.
갈매기 봇으로 구축된 시스템을 자동으로 제거하는 기능을 보장합니다. 하지만 관리자가 직접 역할이나 서버 권한을 변경한 부분이 있다면 해당 부분에 대해서는 직접 수동 제거 작업을 해야 할 수도 있습니다.

  1. 기존의 채널들에서 갈매기 권한을 제거합니다. everyone(모든 사용자)가 메시지를 전송 할 수 있도록 합니다. 다만 관리자만 메시지 전송할 수 있는 채널에는 영향이 없습니다.
  1. 모든 사용자들의 갈매기 역할을 제거합니다. 인증 시스템 구축할 때와는 다르게 반드시 모든 사용자의 역할을 제거해야 합니다. 과도한 api 호출 발생을 우려하여 1명당 1초의 시간이 걸립니다.
  1. 갈매기 역할을 제거합니다.
  1. 캡챠 인증 채널을 제거합니다.
